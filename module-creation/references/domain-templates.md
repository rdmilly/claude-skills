# Domain Templates for Value Chain Decomposition

Pre-built starting chains for common domains. Each template includes the typical stages, transformation statements, and common friction points to probe. Adapt to the user's specific context — these are scaffolding, not prescriptions.

---

## 1. Sales / Lead Pipeline

**Chain:** Discovery → Qualification → Intelligence → Outreach → Response Handling → Conversion → Delivery → Retention → Optimization

| Stage | Transforms | Common Friction |
|-------|-----------|----------------|
| Discovery | Nothing → Known prospect | Manual search, unfocused targeting, no trigger awareness |
| Qualification | Known → Worth pursuing | No scoring criteria, gut-feel prioritization, missing revenue signals |
| Intelligence | Worth pursuing → Understood | Manual research per lead, no competitive context, no pain signal detection |
| Outreach | Understood → Contacted | Generic messaging, single-channel, no personalization, manual sends |
| Response Handling | Contacted → Classified | Manual reply triage, missed follow-ups, no objection tracking |
| Conversion | Classified → Committed | Slow proposal creation, no meeting prep, no objection playbook |
| Delivery | Committed → Served | Scope creep, manual onboarding, no delivery templates |
| Retention | Served → Returning | No check-in system, no upsell triggers, no referral capture |
| Optimization | All stages → Better | No conversion tracking, no A/B testing, no unit economics visibility |

## 2. Content Creation Pipeline

**Chain:** Ideation → Research → Production → Editing → Publishing → Distribution → Engagement → Monetization → Analysis

| Stage | Transforms | Common Friction |
|-------|-----------|----------------|
| Ideation | Nothing → Content concept | Idea drought, no trend awareness, no audience signal reading |
| Research | Concept → Substantiated plan | Manual research, no source aggregation, no fact-checking pipeline |
| Production | Plan → Raw content | Slow creation, no templates, creator bottleneck |
| Editing | Raw → Polished | Manual review, no style consistency, no quality checklist |
| Publishing | Polished → Live | Manual uploads per platform, no scheduling, format conversion |
| Distribution | Live → Seen | Single platform, no cross-posting, no amplification strategy |
| Engagement | Seen → Interacted with | No reply management, no comment tracking, missed interactions |
| Monetization | Interacted → Revenue | No monetization path, no affiliate/sponsor matching |
| Analysis | Revenue → Insights | No performance tracking, no content-to-revenue attribution |

**Video-specific sub-chain:**
Ingest → Transcribe → Tag/Categorize → Edit → Thumbnail → Caption → Publish → Track

## 3. Client / Project Delivery

**Chain:** Scoping → Agreement → Onboarding → Build → Review → Deploy → Training → Support → Upsell

| Stage | Transforms | Common Friction |
|-------|-----------|----------------|
| Scoping | Interest → Defined project | Unclear requirements, scope creep starts here |
| Agreement | Defined → Contracted | Manual proposal writing, slow contract turnaround |
| Onboarding | Contracted → Ready to build | Missing access/credentials, no checklist |
| Build | Ready → Built | Solo builder bottleneck, no progress visibility |
| Review | Built → Approved | Feedback loops too long, unclear revision process |
| Deploy | Approved → Live | Manual deployment, no rollback plan |
| Training | Live → Client can use it | No training materials, no recorded walkthroughs |
| Support | Using → Supported | No ticket system, ad-hoc communication |
| Upsell | Supported → Expanded | No trigger detection, no expansion proposals |

## 4. Infrastructure Management

**Chain:** Provision → Configure → Deploy → Monitor → Alert → Diagnose → Repair → Optimize → Scale

| Stage | Transforms | Common Friction |
|-------|-----------|----------------|
| Provision | Need → Resource exists | Manual server setup, no infrastructure-as-code |
| Configure | Exists → Configured correctly | Manual config, no templates, drift |
| Deploy | Configured → Running | Manual deploys, no CI/CD, downtime |
| Monitor | Running → Observable | Missing metrics, no dashboards |
| Alert | Observable → Noticed when wrong | Alert fatigue, wrong thresholds |
| Diagnose | Noticed → Understood | Manual log reading, no correlation |
| Repair | Understood → Fixed | Manual fixes, no automation |
| Optimize | Fixed → Better | No performance baselines, no cost tracking |
| Scale | Better → Scaled | Manual scaling, no auto-scale rules |

## 5. Product Development

**Chain:** Problem Discovery → Validation → Design → Build → Test → Launch → Measure → Iterate

| Stage | Transforms | Common Friction |
|-------|-----------|----------------|
| Problem Discovery | Nothing → Identified problem | No user research, building for self |
| Validation | Problem → Confirmed demand | No demand testing, no competitor analysis |
| Design | Demand → Spec | No PRD process, spec lives in head |
| Build | Spec → Working code | Solo dev bottleneck, scope creep |
| Test | Code → Verified | No test suite, manual QA |
| Launch | Verified → Available | No launch checklist, no marketing plan |
| Measure | Available → Understood performance | No analytics, no funnel tracking |
| Iterate | Performance → Improved product | No prioritization framework |

## 6. Community / Audience Building

**Chain:** Platform Selection → Content Strategy → Publishing Cadence → Engagement → Growth → Community Formation → Monetization → Sustainability

| Stage | Transforms | Common Friction |
|-------|-----------|----------------|
| Platform Selection | Nothing → Chosen platform(s) | Spreading too thin |
| Content Strategy | Platform → Content plan | No content pillars, reactive posting |
| Publishing Cadence | Plan → Consistent output | Inconsistent posting, burnout |
| Engagement | Output → Two-way interaction | No reply strategy, ignoring comments |
| Growth | Interaction → Expanding reach | No collaboration strategy |
| Community Formation | Reach → Belonging | No community space |
| Monetization | Belonging → Revenue | No offer, no value ladder |
| Sustainability | Revenue → Self-sustaining | Creator burnout, no delegation |

## How to Use These Templates

1. **Start with the closest template** — don't build from scratch if a template fits
2. **Read it with the user** — they'll immediately say "we don't do that" or "we have an extra step"
3. **Customize stages** — add, remove, rename, reorder to match their reality
4. **Then apply the Three Friction Questions** to their customized chain
5. **Cross-reference with other templates** — a sales pipeline often has content creation as a sub-chain within the Outreach stage

Templates 7-9 are intentionally abbreviated. Expand them using the same pattern when needed.
